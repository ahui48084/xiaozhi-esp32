## 2026-06-05 | xiaozhi-server 部署与三件套接入

### 概述

在 RK3566 单板机上完成 `xiaozhi-esp32-server` 的 Docker 部署，接入火山引擎流式 ASR、MiniMax LLM、EdgeTTS，全链路端到端语音对话验证通过。

---

### 环境与架构

**服务器侧**
- 设备：RK3566 单板机（局域网）
- 服务端镜像：`ghcr.nju.edu.cn/xinnan-tech/xiaozhi-esp32-server:server_latest`（版本 0.9.2）
- 服务架构：分离式流水线 `VAD → ASR → LLM → TTS`
  - VAD：SileroVAD
  - ASR：火山引擎流式 ASR（doubao_stream，新版 API Key 认证）
  - LLM：MiniMax（OpenAI 兼容接口）
  - TTS：EdgeTTS

**公网穿透**
- 使用 frp（Fast Reverse Proxy）将 RK3566 内网服务穿透到阿里云 ECS
- WebSocket 服务、OTA 接口各映射一个远端端口

### 已解决的坑

#### 1. `doubao_stream.py` 不支持新版 API Key 认证

**问题**：火山引擎 ASR 控制台已升级为新版，使用 `X-Api-Key` 单一 Key 认证；服务端旧代码只实现了旧版 `X-Api-App-Key` + `X-Api-Access-Key`（AppID + Token）认证，无法对接。

**修复**：在 `core/providers/asr/doubao_stream.py` 中新增 `apikey_auth()` 方法，支持 `auth_method: apikey` 配置项：

```python
def apikey_auth(self):
    return {
        "X-Api-Key": self.api_key,
        "X-Api-Resource-Id": "volc.bigasr.sauc.duration",
        "X-Api-Connect-Id": str(uuid.uuid4()),
    }
```

同时在 `__init__` 中读取 `api_key` 字段，在 `receive_audio` 中按 `auth_method` 分支选择认证方式。

**配置项**（`.config.yaml` 中）：
```yaml
ASR:
  DoubaoStreamASR:
    type: doubao_stream
    auth_method: apikey
    api_key: <火山引擎控制台获取的 API Key>
    appid: ""
    access_token: ""
    cluster: volcengine_streaming_common
    output_dir: tmp/
```

#### 2. `KeyError: 'exit_commands'` 启动崩溃

**问题**：服务端配置加载机制为双文件合并：
- `config.yaml`：**完整默认模板**（含 `exit_commands`、`close_connection_no_voice_time` 等数百个字段）
- `data/.config.yaml`：**用户覆盖配置**（仅写差异项）
- 加载逻辑：`merge_configs(config.yaml, .config.yaml)`

误将挂载的 `config.yaml` 替换为精简版，导致默认字段全部丢失，`ConnectionHandler` 初始化时 `self.config["exit_commands"]` 报 `KeyError`。

**修复**：从镜像原始层提取完整默认 `config.yaml`（约 1149 行），恢复为宿主机挂载文件，所有自定义项仅写入 `.config.yaml`。

**注意**：`.config.yaml` 中 `selected_module` 必须使用**大写命名结构**，小写格式（如 `asr: funlocal`）被合并后不生效：

```yaml
# 正确写法
selected_module:
  VAD: SileroVAD
  ASR: DoubaoStreamASR
  LLM: MinimaxLLM
  TTS: EdgeTTS
  Memory: nomem
  Intent: function_call
```

#### 3. 公网域名访问 HTTP 403（阿里云 ICP 备案拦截）

**问题**：域名注册商为 Cloudflare，未获工信部备案授权，无法在阿里云大陆 ECS 完成 ICP 备案。阿里云对解析到大陆 ECS 的未备案域名的 **明文 HTTP 请求**（检查 `Host` 头）拦截，返回 `Server: Beaver` 的 403 跳转备案页。影响范围：WebSocket 握手（`ws://域名:端口`）和 OTA 接口（`http://域名:端口`）均被拦截。

**临时方案**：`server.websocket` 及测试页 OTA 地址均改为 **ECS 公网 IP** 直连，绕过 Host 检查。

**长期方案**（待定）：
- 方案 A：完成 ICP 备案（域名注册商限制，当前不可行）
- 方案 B：ECS 上配置 nginx + Let's Encrypt 证书，改用 `wss://` 加密传输（明文 HTTP 才被拦截，TLS 加密流量阿里云无法注入跳转，通常放行）
- 方案 C：迁移到香港/海外服务器，无备案拦截

---

#### 4. MiniMax LLM 404 报错

**问题**：OpenAI SDK 使用 `base_url` + `/chat/completions` 拼接最终请求地址。原配置将 `base_url` 写为 MiniMax Native 完整端点（`/v1/text/chatcompletion_v2`），导致实际请求 URL 为 `.../chatcompletion_v2/chat/completions`，返回 404。

**修复**：`base_url` 改为 `https://api.minimax.chat/v1`，SDK 自动拼接为 `https://api.minimax.chat/v1/chat/completions`。

**注意**：LLM 配置使用 `model_name` 字段（非 `model`）。

---

#### 5. 测试页必须通过 HTTP 服务打开

测试页（`test/test_page.html`）在浏览器中通过 `file://` 协议打开时，跨域 `fetch` 请求被浏览器拦截，导致 OTA 连接失败。需通过本地 HTTP 服务访问：

```bash
cd test/
python -m http.server 8006
# 浏览器访问 http://localhost:8006/test_page.html
```

### 关于火山端到端实时语音大模型

当前服务端（0.9.2）架构为分离式流水线，**不支持**火山豆包端到端 RealtimeDialog（全双工语音大模型）。该模型要求音频流直接进出单一模型，与现有 ASR+LLM+TTS 三级流水线架构根本不兼容，暂不接入，保持三件套方案。
